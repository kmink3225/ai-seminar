# qa — 질의응답

## 목적

사용자 질문에 **프로젝트 내부 문서 근거** 를 기반으로 답변한다. 파일은 변경하지 않는다.

## Step

### Step 1 — 쿼리 분해

사용자 쿼리에서 핵심어 2~4 개를 뽑는다. 동의어·영문 용어를 포함한다.

```
예: "리랭킹 언제 필요?"
  → 핵심어: rerank, reranking, 리랭킹
```

### Step 2 — 병렬 Grep 검색

3 개 레이어를 한 번에 Grep 한다 (`output_mode: "content"`, `-n`, `-C 2`).

| 레이어 | 경로 | 우선순위 |
|--------|------|----------|
| L1 변환 레퍼런스 | `references/md/**/*.md` | 1 (주요 근거) |
| L2 주제 문서 | `topics/**/*.md` | 2 |

### Step 3 — 근거 충분성 판정

- **충분** (L1 또는 L2 에서 직접 근거 확보) → Step 4 진행
- **부족** → "내부 문서 근거 부족. 외부 지식 기반 일반 답변이 필요하면 알려 달라." 로 보고하고 종료

### Step 4 — 답변 작성

구조: **결론 → 근거 → 한계**.

```
## 결론
(2~4 문장, 질문에 대한 직접 답)

## 근거
- … (references/md/xxx.md:42)
- … (topics/rag/reranking.md:15)

## 한계
- 내부 문서로 확인되지 않는 부분
- 반대 견해·맥락 의존 요소
```

### Step 5 — 추가 읽을거리 (선택)

관련 내부 문서 2~3 건을 링크로 제시한다.

## 금지

- 근거 없는 추측·일반론
- 외부 지식으로 보강한 내용을 내부 인용처럼 위장
- 질문 재진술로 분량 늘리기
- 전체 파일 Read (Grep 결과 라인만 확인)
