# code — 코드 구현

## 목적

사용자 요구사항 또는 내부 문서의 기법을 **실행 가능한 예제 코드** 로 구현한다.

## 전제

`guides/search.md` 를 먼저 실행해 관련 내부 문서 근거를 확보한다.

## Step

1. **검색** — 관련 기법·API 를 내부 문서에서 찾는다.
2. **요구사항 확정** — 입력·출력·의존성(라이브러리)·런타임을 사용자에게 1 회 확인 (이미 명확하면 생략).
3. **의존성 체크** — 필요한 패키지가 `pyproject.toml` 에 있는지 확인. 없으면 추가 제안 후 사용자 승인.
4. **저장 경로 결정** — 커맨드 인자 또는 아래 기본 규칙.
   - 주제 개념 예제 → `topics/<topic>/<slug>.py` + 동반 `.md` 설명
   - 세미나 데모 → `sessions/<date>-<topic>/demo.py`
5. **구현**
   - 최소 실행 단위로 작성. 불필요한 추상화·에러 핸들링 금지.
   - 파일 상단에 목적·실행법 docstring 1 개.
   - 주석은 WHY 만. WHAT 은 이름으로 표현.
6. **실행 확인 방법 명시** — README 또는 문서 하단에 `python <경로>` 형식으로 명시. 실제 실행은 사용자가 결정.
7. **동반 문서** — 같은 폴더에 `README.md` 또는 동일 slug `.md` 를 만들어 내부 문서로 연결한다 (근거 링크, 파라미터 설명).

## 출력 구조 예

```
topics/agents/tool-use-loop/
├── README.md          # 설명 + 근거 인용
└── tool_use_loop.py   # 실행 가능 코드
```

## 금지

- 근거 없는 임의 API 이름·파라미터 추측
- 실행 불가능한 pseudo-code 를 예제로 제출
- 기존 파일의 변수·함수명 임의 변경
