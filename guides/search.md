# search — 내부 문서 검색

## 목적

`/search [query]` 또는 `/qa`, `/code` 의 근거 수집 단계에서 호출된다. 사용자에게 **관련 문서 경로 + 근거 스니펫** 을 반환한다. 파일은 변경하지 않는다.

## 탐색 범위

| 레이어 | 경로 | 내용 |
|--------|------|------|
| L1 변환된 레퍼런스 | `references/md/**/*.md` | 논문·문서 marker 변환 결과 (주요 출처) |
| L2 주제 문서 | `topics/**/*.md` | 팀이 작성한 주제별 정리 |
| L3 세미나 자료 | `sessions/**/*.md` | 회차별 발표 노트 |
| L4 논문 리뷰 | `papers/**/*.md` | 요약·리뷰 |

## Step

1. **쿼리 분해** — 사용자 쿼리에서 핵심어 2~4 개를 뽑는다. 동의어·영문 용어를 포함한다 (예: "리랭킹" → `rerank|reranking|리랭킹`).
2. **병렬 Grep** — 4 개 레이어를 한 번에 Grep 호출한다 (`output_mode: "content"`, `-n` 활성, `-C 2`).
3. **랭킹** — 매칭 밀도·레이어 우선순위(L1 → L4) 로 정렬한다.
4. **보고** — 상위 5~10 건을 아래 형식으로 출력.

```
[L{1-4}] path/to/file.md:{line}
  > 매칭 스니펫 한두 줄
```

5. **요약 1 줄** — 검색 결과가 커버하는 범위를 한 문장으로 평가 ("주제 X 관련 문서 N 건, 대부분 L1 레퍼런스" 등).

## 금지

- 근거 없는 추측·일반론 답변
- 전체 파일 Read (Grep 결과 라인만 확인)
