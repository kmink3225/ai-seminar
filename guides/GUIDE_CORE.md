# GUIDE_CORE — 공통 규칙

모든 슬래시 커맨드에서 작업 시작 전 먼저 로드한다.

---

## 1. 문서 작성 규칙

- **한다 체** — `~한다/~이다/~된다`. `~합니다/~입니다` 금지.
- **이모지 금지** — 마크다운 본문·코드·커밋 메시지 어디에도 이모지를 넣지 않는다.
- **섹션 번호 수동 부여 허용** — 이 프로젝트는 plain Markdown 아카이브다. `## 1. 개요` 처럼 직접 번호를 붙여도 된다. H2 가 3 개 이상인 긴 문서는 네비게이션·인용성을 위해 번호 부여를 권장한다. (Quarto 블로그 프로젝트와 달리 자동 번호 매기기가 없다.)
- **제목 계층 건너뛰기 금지** — `#` → `##` → `###` 순서로 사용한다. `##` 없이 `###` 부터 시작 금지.
- **코드 펜스 언어 식별자** — 코드 블록에 언어를 명시한다 (```` ```python ````, ```` ```bash ````, ```` ```yaml ````). 언어가 없으면 ```` ```text ```` 또는 식별자 없이 둔다.
- **파일명** — `kebab-case.md`. 한글·공백·대문자 금지.
- **메타데이터** — 새 문서 상단에 YAML 프론트매터 권장:
  ```yaml
  ---
  title: ...
  author: ...
  date: YYYY-MM-DD
  tags: [...]
  source: (원본 PDF 경로 또는 URL)
  ---
  ```

---

## 2. 검색/조회 규칙

- 내부 문서 검색은 `references/md/`, `topics/` 2개 폴더를 **병렬 Grep** 으로 조회한다.
- 근거가 없는 주장은 하지 않는다. 근거가 부족하면 "근거 부족" 또는 "모름" 으로 명시한다.
- 인용 시 파일 경로와 라인 번호를 `path:line` 형식으로 남긴다.

---

## 3. 파일 수정 규칙

- **수정 전 반드시 Read 도구로 파일 내용을 확인**한다. 추측으로 작성 금지.
- **기존 이름(변수·함수·파일) 보존** — 리팩터링 시 임의 변경 금지. 변경이 필요하면 사용자 확인 후 진행.
- **로직 축약 금지** — 동작하는 코드 구조를 요청 범위 밖에서 임의로 바꾸지 않는다.
- `references/md/` 하위 파일은 **자동 생성물** — 직접 수정하지 않는다. 수정이 필요하면 원본 PDF 를 교체하고 재변환한다.

---

## 4. 저장 위치 규칙

새로 만든 산출물은 아래 기준으로 배치한다.

| 산출물 | 저장 위치 | 비고 |
|--------|-----------|------|
| 개념 본문 (정의·원리·예제·Q&A) | `topics/{주제}/` | 발표용 문서, 세미나 Q&A·논의도 여기에 둔다 |
| 논문 원본 PDF | `references/pdfs/` | marker 로 `references/md/` 자동 변환 |
| 외부 링크 모음 | `references/README.md` 의 링크 섹션 | |
| 실행 가능한 예제 코드 | 관련 `topics/*/` 내부 | |

## 5. Self-Check

작업 보고 전 아래 항목을 **출력**한다.

```
Self-Check:
- [ ] 한다 체 준수
- [ ] 이모지 없음
- [ ] 제목 계층·코드 펜스 언어 식별자 준수
- [ ] 근거 인용 (path:line)
- [ ] 파일 저장 위치 규칙 준수
```

## 6. 커밋·푸시

- 사용자가 명시적으로 요청할 때만 커밋·푸시한다.
- 커밋 메시지는 한국어 기반, 변경 의도(why) 중심. 이모지 금지.

## 7. PDF 변환 운영 규칙 (배치 권고)

marker 변환은 **모델 로딩·추론에 수 분 ~ 수십 분** 이 걸린다. 대화 흐름 중 실시간으로 돌리면 작업이 막힌다. 따라서 **배치 처리** 를 원칙으로 한다.

### 권장 운영 방식

1. 관심 PDF 를 발견하면 즉시 변환하지 말고 `references/pdfs/` 에 **모아 둔다** (파일 복사만).
2. **점심시간·퇴근 전·주말** 등 장시간 자리를 비우는 타이밍에 한 번에 돌린다.
   ```bash
   python system-scripts/convert_pdfs.py
   ```
3. 신규 투입 PDF 가 3 개 이상 쌓였거나 마지막 변환 이후 24 시간 이상 지났을 때가 좋은 배치 시점.

### 에이전트 행동 규칙

사용자가 **PDF 변환·marker 실행·`/qa` 대상 확장** 을 요청하거나, 현재 세션에서 `references/pdfs/` 에 신규 PDF 를 추가하는 행동이 관찰되면, 에이전트는 즉시 변환을 시도하기 전에 **아래 권고 메시지를 먼저 출력** 한다.

```
[PDF 변환 권고] marker 변환은 수 분 이상 걸리는 배치 작업입니다.
지금 바로 돌리기보다 다음 타이밍에 한 번에 처리하기를 권장합니다:
  - 점심시간 시작 직전 또는 퇴근 직전
  - 신규 PDF 가 3 개 이상 쌓였을 때
  - 마지막 변환 이후 24 시간 이상 지났을 때
지금 바로 실행이 필요하면 "지금 변환" 이라고 답해 주세요.
```

- 사용자가 "지금 변환" / "배치 무시" / "바로 돌려" 등으로 **명시적 동의** 를 하면 즉시 실행.
- 동의가 없으면 `references/pdfs/` 에만 추가하고 변환은 보류한다.
