# References

외부 참고자료 아카이브. PDF는 marker로 Markdown 변환 후 검색/조회/분석에 사용.

## 구조

```
references/
├── pdfs/   # 원본 PDF 투입 위치
└── md/     # marker 변환 결과 (자동 생성)
    └── <pdf_stem>/
        ├── <pdf_stem>.md
        └── <이미지/메타>
```

## 사용 흐름 (배치 원칙)

marker 변환은 수 분 이상 걸리는 배치 작업이다. **실시간 변환을 피하고 모아서 한 번에 처리**한다.

1. 새 PDF 를 `references/pdfs/` 에 넣어 둔다 (하위 폴더 가능). 변환은 하지 않는다.
2. 아래 타이밍에 **한 번에** 변환한다.
   - 점심시간 시작 직전 또는 퇴근 직전
   - 신규 PDF 가 3 개 이상 쌓였을 때
   - 마지막 변환 이후 24 시간 이상 지났을 때
3. 프로젝트 루트에서 변환 스크립트 실행.

```bash
python system-scripts/convert_pdfs.py
```

- 이미 변환된 파일은 자동으로 건너뜀 (idempotent)
- 실패한 파일은 요약에 표시됨

> 에이전트는 대화 중 변환 요청을 받으면 즉시 실행하지 않고 `guides/GUIDE_CORE.md` §7 의 배치 권고 메시지를 먼저 출력한다.
